# Anomaly Analysis

## Insert Anomaly
In the given dataset, a new product cannot be added unless an order exists. This creates difficulty in inserting independent product data.

## Update Anomaly
Customer details are repeated multiple times. If a customer's information is updated in one row but not in others, it leads to data inconsistency.

## Delete Anomaly
Deleting an order may result in loss of important customer information, which should ideally be stored separately.
